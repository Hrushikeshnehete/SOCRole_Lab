# Overview

This repository documents my work on the SOC Role in Blue Team lab from TryHackMe. The exercise focuses on understanding how a Security Operations Center (SOC) functions, including analyst responsibilities, threat monitoring, and incident handling procedures.

# Objectives

Understand SOC structure and team roles.
Learn how to detect, analyze, and respond to cyber incidents.
Explore the workflow of security analysts using SIEM tools.
Strengthen Blue Team defensive skills.

# Key Learnings

SOC Hierarchy:

Tier 1 (Alert Analyst): Monitors alerts and identifies potential threats.
Tier 2 (Incident Responder): Investigates and mitigates confirmed incidents.
Tier 3 (Threat Hunter): Proactively searches for undetected threats.
Core Concepts Covered:
Real-time alert monitoring
Log correlation and analysis
Threat intelligence application
Incident response processes
Documentation and escalation workflow
